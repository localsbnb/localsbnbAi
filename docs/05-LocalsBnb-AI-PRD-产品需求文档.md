## 二、产品背景与目标

### 2.1 产品背景

路客云是面向民宿/酒店经营者的多平台 PMS。LocalsBnb AI 将路客云核心能力封装为 MCP，使店长、运营、老板等可在任意支持 MCP 的 AI 终端（如 Claude Desktop、Cursor、钉钉/飞书 Bot）中，通过自然语言查询经营数据、待办事项，并完成入住/退房、发指引、开关房态、改价等操作。

### 2.2 产品目标

- **核心目标**：在保证安全与合规的前提下，提供「查 + 办」的完整能力，一次配置、多端复用。
- **业务目标**：提升路客云用户粘性，形成可收费的增值能力（模式 B：按次/月收费），为后续多系统统一 AI 入口打基础。
- **体验目标**：响应稳定、错误提示拟人化、写操作有明确确认步骤，数据输出脱敏。

### 2.3 目标用户与优先级

| 优先级 | 角色 | 典型场景 | 核心诉求 |
|--------|------|----------|----------|
| P0 | 店长 | 日常查房态、办入住退房、发密码与指引 | 高效、少登系统 |
| P1 | 运营 | 查经营数据、多日报表、房态与房价 | 数据及时、可汇总 |
| P2 | 老板 | 看经营概况、异常与待办，偶尔下发指令 | 一目了然、可授权 |
| P3 | 开发者 | 集成到自建 Bot、自定义工作流 | 协议开放、可配置 |

## 三、范围与边界

### 3.1 在范围内

- 基于 token + campId 的鉴权与初始化。
- 待办（预抵/预离/在住）、订单查询与详情、经营数据（基础/多日）、房态、房价、可售情况等**查询能力**。
- 订单办理入住/退房、发送入住指引、发送密码；房态开/关；改房价等**操作能力**（均需 Human-in-the-loop）。
- 数据脱敏、写操作确认流程、异常友好提示、分页与截断策略。
- 模型接入与配置（国内 DeepSeek、海外 GPT，营地/用户可选覆盖）。

### 3.2 不在范围内（本期）

- 路客云 PC/App 原有功能的改造（仅通过 MCP 暴露能力）。
- 创建订单、取消订单等部分写操作可在后续迭代纳入，本期以「待办+入住退房+发指引/密码+房态+改价」为主。
- 多租户/多 camp 在同一会话内的切换交互细节（可后续定义「当前 camp」切换）。
- 用户自带 API Key（模式 A）的完整产品化与计费，本期以模式 B 为主。

## 四、功能需求详述

### 4.1 鉴权与初始化

#### 4.1.1 初始化配置

| 属性 | 说明 |
|------|------|
| 功能描述 | 用户或管理员在 AI 客户端配置 MCP 时，需提供 token 与 campId，用于标识身份与数据范围。 |
| 优先级 | P0 |
| 输入 | **token**（必填）：路客云侧颁发的访问凭证。<br>**campId**（必填）：营地/物业 ID，与 token 绑定。 |
| 业务规则 | 1. token 与 campId 必须成对有效，否则禁止调用任何工具。<br>2. 所有 MCP 请求（工具调用）必须携带该 token 与 campId；由 MCP Server 在协议层注入或校验。<br>3. 权限范围（scope）由路客云根据 token/camp 解析，MCP 层做转发与校验。 |
| 验收标准 | 1. 配置正确时，工具可正常调用路客云 API。<br>2. token 无效或与 campId 不匹配时，返回明确鉴权错误，不调用下游 API。<br>3. 文档与配置示例中明确说明 token、campId 的获取方式（路客云后台或管理员分配）。 |

#### 4.1.2 权限范围（Scope）

| Scope | 说明 | 涉及工具（示例） |
|-------|------|------------------|
| orders:read | 订单查询 | query_orders, get_order_details, 待办类查询 |
| orders:write | 订单写操作 | check_in_order, check_out_order, send_guide, send_password |
| rooms:read | 房态/房价查询 | query_room_status, query_room_prices, query_availability |
| rooms:write | 房态/房价写操作 | update_room_status, update_room_price, batch_update_prices |
| finance:read | 经营/财务数据查询 | query_operational_data, query_financial_data, analyze_revenue |
| channels:read | 渠道查询 | list_channels, get_channel_details |
| * | 全部权限 | 仅管理员或特殊 token |

### 4.2 待办事项（预抵 / 预离 / 在住）

#### 4.2.1 查询预抵订单

| 属性 | 说明 |
|------|------|
| 功能描述 | 查询指定日期范围内「待入住」的订单（预抵），支持分页。 |
| 优先级 | P0 |
| MCP 工具名 | 建议 `query_todos_pre_arrival` 或复用 `query_orders` 且 status=pending + 日期为入住日。 |
| 输入 | **campId**（必填，来自上下文）。<br>**storeId**（必填，可与 campId 一致或由后端映射）。<br>**date**（必填，YYYY-MM-DD，查询该日的预抵）。<br>**page**（可选，默认 1）。<br>**pageSize**（可选，默认 20，最大 100）。 |
| 输出 | 订单列表：orderId、orderNo、guestName（脱敏）、phone（脱敏）、roomType、checkInDate、checkOutDate、status 等；total、page、pageSize。 |
| 业务规则 | 1. 仅返回「入住日期在查询范围内且状态为待入住」的订单。<br>2. 列表中的姓名、手机号、证件号须脱敏。<br>3. 强制分页，单页最多 100 条。 |
| 验收标准 | 1. 返回结果与路客云后台「预抵」视图一致（以路客云 API 为准）。<br>2. 脱敏规则符合 5.2 节。 |

#### 4.2.2 查询预离订单

| 属性 | 说明 |
|------|------|
| 功能描述 | 查询指定日期范围内「待退房」的订单（预离），支持分页。 |
| 优先级 | P0 |
| MCP 工具名 | 建议 `query_todos_pre_departure` 或复用 `query_orders` 且 status=checked_in + 日期为退房日。 |
| 输入 | **campId** / **storeId**、**date**（退房日）、**page**、**pageSize**。 |
| 输出 | 同 4.2.1 结构，状态为在住、退房日在查询日。 |
| 业务规则 | 同 4.2.1，逻辑为「退房日 = date 且 status=checked_in」。 |
| 验收标准 | 与路客云「预离」逻辑一致，脱敏生效。 |

#### 4.2.3 查询在住订单

| 属性 | 说明 |
|------|------|
| 功能描述 | 查询当前在住订单列表，支持分页。 |
| 优先级 | P0 |
| MCP 工具名 | 建议 `query_todos_in_house` 或 `query_orders` 且 status=checked_in。 |
| 输入 | **campId** / **storeId**、**page**、**pageSize**。 |
| 输出 | 在住订单列表，字段同 4.2.1。 |
| 业务规则 | 仅返回 status=checked_in。 |
| 验收标准 | 与路客云「在住」一致，脱敏生效。 |

### 4.3 订单查询

#### 4.3.1 查询订单列表（query_orders）

| 属性 | 说明 |
|------|------|
| 功能描述 | 按条件查询订单列表，支持日期范围、状态、分页等。 |

#### 4.3.2 获取订单详情（get_order_details）

|------|------|

| 输入 | orderId（必填）。 |

| 业务规则 | 1. 仅能查询当前 token/campId 权限范围内的订单。<br>2. 若订单不存在或无权限，返回明确错误。 |
| 验收标准 | 与《03-工具定义规范》2.2 一致，且脱敏。 |

### 4.4 订单写操作（均需 Human-in-the-loop）

#### 4.4.1 办理入住（check_in_order）

| 属性 | 说明 |
|------|------|
| 功能描述 | 为指定订单办理入住。执行前必须经用户确认。 |
| 优先级 | P0 |
| 输入 | orderId（必填）、roomId（可选，若已排房）。 |
| 确认流程 | 1. AI 首次调用时，MCP 返回「待确认」结构：操作描述（如「为订单 XXX 办理入住」）、orderId、需用户确认。<br>2. 用户明确确认后，AI 再次调用并传入 confirm=true 或调用 execute_check_in，MCP 才调用路客云写接口。<br>3. 执行成功后返回订单新状态。 |
| 业务规则 | 1. 订单状态必须为待入住。<br>2. 同一订单不可重复入住。<br>3. 操作需落审计日志（谁、何时、哪笔订单）。 |
| 验收标准 | 1. 未确认不执行；确认后执行结果与路客云一致。<br>2. 审计可查。 |

#### 4.4.2 办理退房（check_out_order）

| 属性 | 说明 |
|------|------|
| 功能描述 | 为指定订单办理退房。执行前必须经用户确认。 |
| 优先级 | P0 |
| 输入 | orderId（必填）。 |
| 确认流程 | 同 4.4.1，先返回待确认，用户确认后再执行。 |
| 业务规则 | 订单状态必须为在住；不可重复退房；审计落库。 |
| 验收标准 | 同 4.4.1。 |

#### 4.4.3 发送入住指引（send_guide / 或路客云现有接口名）

| 属性 | 说明 |
|------|------|
| 功能描述 | 向指定订单的客人发送入住指引（如短信/模板消息）。执行前必须经用户确认。 |
| 优先级 | P1 |
| 输入 | orderId（必填）；可选：templateId、渠道等（以路客云 API 为准）。 |
| 确认流程 | 同 4.4.1。 |
| 业务规则 | 依赖路客云现有「发入住指引」能力；若暂无独立接口，可标注为后续迭代。 |
| 验收标准 | 确认后发送成功，且可审计。 |

#### 4.4.4 发送密码（send_password / 或路客云现有接口名）

| 属性 | 说明 |
|------|------|
| 功能描述 | 向指定订单的客人发送门锁/房门密码。执行前必须经用户确认。 |
| 优先级 | P1 |
| 输入 | orderId（必填）；可选：渠道（以路客云 API 为准）。 |
| 确认流程 | 同 4.4.1。 |
| 业务规则 | 依赖路客云现有「发密码」能力。 |
| 验收标准 | 确认后发送成功，且可审计。 |

### 4.5 经营数据查询

#### 4.5.1 查询经营数据（query_operational_data）

| 属性 | 说明 |
|------|------|
| 功能描述 | 查询指定日期范围内的经营统计：订单数、收入、入住率、RevPAR 等；支持按日明细。 |

#### 4.5.2 多日经营数据与汇总

| 属性 | 说明 |
|------|------|

## 五、非功能需求

### 5.1 性能与可用性

| 需求项 | 说明 |
|--------|------|
| 响应时间 | 单次工具调用（不含大模型推理）P95 ≤ 5s；路客云 API 超时建议 10–15s，超时后进入 5.4 异常话术。 |
| 可用性 | 依赖路客云 API 与 MCP Server 部署可用性；目标与路客云 SLA 对齐。 |
| 分页与截断 | 所有列表类接口强制分页，pageSize 最大 100；经营数据 dailyData 等可限制条数（如 31）；单条详情与汇总文本长度可设上限，避免 AI Context 溢出。 |

### 5.2 数据脱敏与隐私

| 需求项 | 说明 |
|--------|------|
| 脱敏范围 | 所有返回给 AI 客户端（进而可能展示给用户）的数据中，**姓名、手机号、证件号**必须脱敏。 |
| 脱敏规则 | **姓名**：保留姓，其余用 * 替代（如「张**」）。**手机号**：保留前 3 后 4，中间用 ****（如 138****1234）。**证件号**：仅保留后 4 位或全部用 *（可配置）。 |
| 实施位置 | 在 MCP Server 或统一网关层，在返回给 AI 之前对结构化结果与文本做脱敏，不依赖 AI 侧。 |
| 合规 | 符合最小必要原则；若后续有「申请查看明文」流程，需单独设计权限与审计。 |

### 5.3 写操作与 Human-in-the-loop

| 需求项 | 说明 |
|--------|------|
| 适用操作 | 办理入住、办理退房、发送入住指引、发送密码、开/关房态、改房价（含批量）。 |
| 流程 | 1）用户表达意图 → 2）AI 调用 MCP 工具 → 3）MCP 不直接执行，返回「待确认」结构（含操作描述、主键）→ 4）AI 向用户展示并询问确认 → 5）用户确认后，AI 再次调用并带确认标识 → 6）MCP 调用路客云写接口 → 7）返回执行结果。 |
| 确认方式 | 支持自然语言确认（如「确认」「执行」）或客户端按钮；MCP 层通过参数（如 confirm=true）区分「仅预览」与「确认执行」。 |
| 审计 | 谁、何时、对哪笔订单/哪间房/哪日房价执行了何操作，在路客云侧落库并可查。 |

### 5.4 异常处理与友好提示

| 场景 | 返回策略 |
|------|----------|
| 鉴权失败 | 明确提示「未授权」或「token/营地无效」，不暴露内部信息。 |
| 路客云 API 超时 | 拟人化话术，如：「老板，现在的网络有点开小差，我没能拿到最新的房态，您稍等一分钟我再试。」 |
| 限流 | 「请求有点多，稍等再试。」 |
| 部分失败 | 「已拿到 XX，但 YY 暂时没拿到，您可以先看 XX。」 |
| 参数错误 | 明确说明缺少或错误的参数，便于用户或 AI 修正。 |

话术模板见附录 B。

### 5.5 模型接入与配置（模式 B）

| 需求项 | 说明 |
|--------|------|
| 默认策略 | 国内部署默认 DeepSeek，海外默认 GPT；API Key 由路客云统一管理。 |
| 营地级 | 可为单个 camp 指定模型（如 GPT-4o）或「自动」沿用地域默认。 |
| 用户级 | 可选，仅在对开发者/高级用户开放时提供。 |
| 扩展 | 新增模型时通过配置与适配器扩展，不改业务代码。 |

## 六、数据与接口规范

### 6.1 分页规范

| 参数 | 类型 | 默认值 | 最大值 | 说明 |
|------|------|--------|--------|------|
| page | number | 1 | - | 页码，从 1 开始 |
| pageSize | number | 20 | 100 | 每页条数 |

所有列表类响应包含：list、total、page、pageSize。

### 6.2 日期规范

- 格式：YYYY-MM-DD。
- 日期范围：单次查询建议不超过 90 天（订单/经营）；房态/房价建议不超过 31 天；超出时可拒绝或截断并提示。

### 6.3 统一错误响应结构

```json
{
  "code": "非 0 错误码",
  "message": "用户或 AI 可读的提示",
  "friendlyMessage": "可选，拟人化话术，用于直接回复用户"
}
```

常见错误码见附录 A。

### 6.4 写操作「待确认」响应结构

```json
{
  "requireConfirmation": true,
  "action": "check_in_order",
  "summary": "为订单 12345 办理入住",
  "params": { "orderId": "12345" },
  "message": "请确认是否执行上述操作。"
}
```

用户确认后，AI 再次调用时传入上述 params 并增加 confirm: true（或由服务端约定字段），MCP 再执行写接口。

## 七、验收标准汇总

### 7.1 按模块

| 模块 | 验收要点 |
|------|----------|
| 鉴权与初始化 | token+campId 正确可调通；错误时返回明确鉴权失败且不调下游。 |
| 待办 | 预抵/预离/在住三类查询结果正确、分页与脱敏生效。 |
| 订单查询 | 列表与详情与路客云一致，脱敏生效，分页有效。 |
| 订单写操作 | 入住/退房/发指引/发密码均需确认后执行，审计可查。 |
| 经营数据 | 汇总与多日数据正确，截断策略生效。 |
| 房态 | 查询正确；开/关房态需确认后执行，审计可查。 |
| 房价 | 查询与可售正确；改价需确认后执行，审计可查。 |
| 异常 | 超时/限流等返回友好话术，符合 5.4。 |
| 模型配置 | 地域/营地配置生效，国内 DeepSeek、海外 GPT 可切换。 |

### 7.2 端到端场景

1. **店长查今日预抵并办入住**：查询今日预抵 → 选择订单 → 办理入住 → 确认 → 执行成功并提示。
2. **运营查近 7 天经营数据**：输入 7 天范围 → 返回汇总与每日明细（或截断提示）→ 无敏感信息。
3. **老板关某房型明日房态**：请求关房态 → 返回待确认 → 确认 → 执行成功并提示。

### 附录 A：错误码

| 错误码 | 说明 | 友好话术建议 |
|--------|------|--------------|
| AUTH_REQUIRED | 未提供 token/campId | 请先配置 token 与营地信息。 |
| AUTH_INVALID | token 或 campId 无效 | 当前登录已失效，请重新配置。 |
| PERMISSION_DENIED | 无该操作权限 | 您没有执行该操作的权限。 |
| INVALID_PARAMS | 参数错误 | 请检查输入条件后重试。 |
| API_ERROR | 路客云 API 业务错误 | 根据 API 消息返回或使用通用「操作失败，请稍后重试」。 |
| API_TIMEOUT | 路客云 API 超时 | 老板，网络有点开小差，稍等一分钟再试。 |
| API_RATE_LIMIT | 请求过于频繁 | 请求有点多，稍等再试。 |
| INTERNAL_ERROR | 服务内部错误 | 系统暂时有点忙，请稍后再试。 |
