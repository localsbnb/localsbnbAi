# PRD 参考模板

## 文档结构总览

```
1. 文档信息
2. 术语与缩写
3. 产品背景与目标
   - 产品背景
   - 产品目标
   - 目标用户与优先级
4. 范围与边界
   - 在范围内
   - 不在范围内
5. 功能需求详述
   - 按模块划分（鉴权、查询、写操作等）
   - 每功能：功能描述、优先级、输入、输出、业务规则、验收标准
6. 非功能需求
   - 数据脱敏、权限、异常处理、分页规范等
7. 验收标准汇总
8. 附录
   - 错误码、话术模板、接口约定等
```

## 功能需求表格模板

```markdown
#### X.X.X 功能名称

| 属性 | 说明 |
|------|------|
| 功能描述 | 一句话描述 |
| 优先级 | P0/P1/P2 |
| 输入 | **param1**（必填）：说明。<br>**param2**（可选）：说明。 |
| 输出 | 返回字段列表及说明 |
| 业务规则 | 1. 规则一<br>2. 规则二 |
| 验收标准 | 1. 可验证点一<br>2. 可验证点二 |
```

## 写操作确认流程（Human-in-the-loop）

1. AI 首次调用 → 返回「待确认」结构（action、summary、params）
2. 用户明确确认
3. AI 再次调用并传入 confirm=true
4. 服务端执行并返回结果
5. 操作落审计日志

## 分页规范

| 参数 | 类型 | 默认值 | 最大值 |
|------|------|--------|--------|
| page | number | 1 | - |
| pageSize | number | 20 | 100 |

响应包含：list、total、page、pageSize。

## 日期规范

- 格式：YYYY-MM-DD
- 单次查询建议不超过 90 天（订单/经营）；房态/房价不超过 31 天
