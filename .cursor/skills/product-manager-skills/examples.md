# 示例

## 功能需求示例

```markdown
#### 4.2.1 查询预抵订单

| 属性 | 说明 |
|------|------|
| 功能描述 | 查询指定日期范围内「待入住」的订单（预抵），支持分页。 |
| 优先级 | P0 |
| 输入 | **campId**（必填，来自上下文）。<br>**date**（必填，YYYY-MM-DD）。<br>**page**（可选，默认 1）。<br>**pageSize**（可选，默认 20，最大 100）。 |
| 输出 | 订单列表：orderId、orderNo、guestName（脱敏）、roomType、checkInDate、status 等；total、page、pageSize。 |
| 业务规则 | 1. 仅返回「入住日期在查询范围内且状态为待入住」的订单。<br>2. 姓名、手机号须脱敏。<br>3. 强制分页，单页最多 100 条。 |
| 验收标准 | 1. 返回结果与后台「预抵」视图一致。<br>2. 脱敏规则生效。 |
```

## 用户故事示例

**示例 1**
- 作为店长，我希望通过自然语言查询今日预抵订单，以便快速了解待办而不必登录系统。
- Given：已配置 token 与 campId
- When：用户询问「今天有哪些预抵」
- Then：返回今日待入住订单列表，敏感信息脱敏

**示例 2**
- 作为运营，我希望查看近 7 天经营数据汇总，以便快速掌握营收与入住率。
- Given：有 orders:read 或 finance:read 权限
- When：用户询问「最近 7 天经营情况」
- Then：返回订单数、收入、入住率、RevPAR 等汇总及每日明细

## 目标用户优先级表示例

| 优先级 | 角色 | 典型场景 | 核心诉求 |
|--------|------|----------|----------|
| P0 | 店长 | 查房态、办入住退房、发指引 | 高效、少登系统 |
| P1 | 运营 | 查经营数据、多日报表 | 数据及时、可汇总 |
| P2 | 老板 | 看经营概况、异常与待办 | 一目了然、可授权 |
